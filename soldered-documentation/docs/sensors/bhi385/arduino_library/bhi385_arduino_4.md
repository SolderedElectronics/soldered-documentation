This page contains some tips in case you are having problems using this product.

<ExpandableSection title="My sensor won't initialize!">

#### Check the Qwiic cable
Ensure that your Qwiic cable is properly connected and in good condition. Try using the same cable with another Qwiic-compatible device to verify it works.

#### Check I2C pins
If you are connecting over bare I2C pins (not Qwiic), double-check that SDA and SCL are correctly wired. Different microcontrollers have different default I2C pins.

#### Verify the I2C address
The Soldered BHI385 board ships with **JP5 open**, which sets the I2C address to **0x29** (`BHI385_I2C_ADDR_HIGH`). Make sure you are calling:
```cpp
imu.begin(BHI385_I2C_ADDR_HIGH);
```
If you have bridged JP5, use `BHI385_I2C_ADDR_LOW` (0x28) instead.

#### Scan for I2C devices
Run an [**I2C scanner sketch**](https://github.com/SolderedElectronics/Soldered-Hacky-Codes/tree/main/I2C_Scanner) to check whether the sensor is visible on the bus. The BHI385 should appear at **0x28** or **0x29** depending on JP5.

#### Check power
The BHI385 chip requires a stable **1.8V supply** generated by the onboard LDO. The LDO is powered by the **3.3V** pin. Make sure:
- The board is connected to a 3.3V supply (not 5V directly to the 3V3 pin).
- JP3 is **closed** (the LDO enable jumper must remain bridged for normal operation).

</ExpandableSection>

<ExpandableSection title="Firmware upload fails or takes very long">

#### Use 400 kHz I2C
At the default 100 kHz I2C speed, firmware upload takes approximately **5 seconds**. Add this line before `imu.begin()`:
```cpp
Wire.setClock(400000);
```
This reduces upload time to approximately **1.3 seconds**.

#### Check power supply stability
The firmware upload transfers a large binary over I2C. If the 3.3V supply is noisy or droops during transfer, the upload will fail. Try a different USB cable or power supply.

#### Enable debug output
Call `imu.enableDebug()` after `begin()` to print step-by-step upload progress to Serial:
```cpp
imu.begin(BHI385_I2C_ADDR_HIGH);
imu.enableDebug();
imu.loadFirmware(bhi385Firmware, sizeof(bhi385Firmware));
```
This will print exact I2C error codes if something fails.

#### Verify the firmware file
Make sure `BHI385_firmware.h` is in your sketch folder and `#include "BHI385_firmware.h"` is at the top of your sketch. The firmware variable must be named `bhi385Firmware`.

</ExpandableSection>

<ExpandableSection title="Sensor reads zeros or incorrect data">

#### Enable the sensor before reading
You must call `enableAccelerometer()` or `enableGyroscope()` (after `loadFirmware()`) before any data is available. Skipping these calls will result in all getters returning zero.

#### Call update() in every loop iteration
The BHI385 uses a FIFO-based protocol. Data is only parsed when you call `imu.update()`. Ensure it is called regularly (at least every 20–50 ms for 100 Hz sensors):
```cpp
void loop()
{
    if (imu.update())
    {
        if (imu.accelUpdated())
        {
            // read data here
        }
        imu.clearUpdatedFlags();
    }
    delay(20);
}
```

#### Check the updated flags
Always verify `accelUpdated()` / `gyroUpdated()` before reading values. If the FIFO contained no new data for that sensor in the last `update()` call, the previous value remains but the flag is false.

#### Verify the dynamic range setting
If acceleration values seem clipped at a round number (e.g. always exactly 8.0 g), the full-scale range may be too small. Increase it:
```cpp
imu.enableAccelerometer(100.0f, BHI385_ACCEL_32G);
```

</ExpandableSection>

<ExpandableSection title="Interrupt pin not triggering">

#### Check INT pin wiring
The **INT** pin on the breakout board is the active-high interrupt output from the BHI385. Connect it to a GPIO pin on your microcontroller that supports external interrupts.

#### Use RISING edge
Configure the interrupt to trigger on a **rising edge**:
```cpp
attachInterrupt(digitalPinToInterrupt(INT_PIN), onInterrupt, RISING);
```

#### See the interrupt examples
The library includes interrupt-driven examples for the most common use cases:
