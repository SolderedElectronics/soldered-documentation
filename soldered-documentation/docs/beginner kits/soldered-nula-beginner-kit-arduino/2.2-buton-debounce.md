Cilj ovog primjera je pokazati kako učiniti pritiske tipkala **pouzdanijima i ujednačenima** korištenjem tehnike koja se zove **debounce**.  
Kad pritisnemo fizičko tipkalo, njegovi kontakti mogu kratko poskakivati, pa mikrokontroler umjesto jednog pritiska vidi nekoliko.  
U ovom primjeru koristit ćemo **softverski debounce** uz pomoć funkcije `millis()` kako bismo postigli da jedan pritisak tipkala znači jedno paljenje ili gašenje LED diode.

<SuccessBox>
**U ovoj dokumentaciji naučit ćeš:**
    - Kako čitati stanje tipkala funkcijom `digitalRead()`.
    - Što je **debounce tipkala** i kako ga riješiti koristeći `millis()`.
    - Kako na svaki pritisak tipkala promijeniti stanje LED diode.
</SuccessBox>

**Potreban hardver:**
    - 1x Soldered NULA MINI pločica
    - 1x breadboard
    - 1x tipkalo 
    - 1x LED dioda (bilo koje boje)
    - 1x otpornik od 330 Ω
    - 1x otpornik od 10 kΩ

    - USB-C kabel

Ovaj primjer započinjemo spajanjem svih komponenti. Samo prati korake u nastavku i sve će proći bez puno muke.

### 1. Postavi NULA MINI pločicu na breadboard

<InfoBox>Za ovaj korak trebaš poznavati unutarnju strukturu breadboarda i njegove naponske vodove. Ako ti to još nije poznato, pogledaj [**PLACEHOLDER**] stranicu dokumentacije.</InfoBox>

Ako tvoja pločica ima zalemljene muške konektore na pinovima, preporučujemo da je postaviš na breadboard, jer će ti spajanje ostalih komponenti biti lakše i urednije.

<InfoBox>  
### Što je pull-down otpornik?  

Kada tipkalo spojiš izravno na pin **NULA MINI** pločice, pin ne zna uvijek treba li čitati stanje **HIGH** ili **LOW** dok tipkalo nije pritisnuto. U tom „lebdećem“ stanju pin može hvatati električni šum i nasumično prelaziti između HIGH i LOW, što dovodi do nepredvidivog ponašanja.  

**Pull-down otpornik** rješava taj problem tako da lagano „povlači“ vrijednost pina dolje na **LOW (0 V)** kada tipkalo nije pritisnuto. Možeš ga zamisliti kao oprugu koja uvijek vraća pin na masu kada ništa drugo ne djeluje na njega.  

- Kada tipkalo **nije pritisnuto**, otpornik spaja pin na **GND**, pa NULA pločica očitava **LOW**.  
- Kada je tipkalo **pritisnuto**, tipkalo spaja pin izravno na **3.3 V**, i pločica očitava **HIGH**.  

Vrijednost otpornika (najčešće **10 kΩ**) bira se tako da bude dovoljno jak da drži pin u stanju LOW kada je sklop u mirovanju, a dovoljno slab da ne ometa struju kad je tipkalo pritisnuto.  

Bez pull-down otpornika ulaz s tipkala može se ponašati nepredvidivo i ponekad pokazivati HIGH čak i kada tipkalo ne pritišćeš.  
</InfoBox>

## Čitanje stanja tipkala

Za čitanje stanja tipkala koristimo funkciju `digitalRead()`. Važno je da je pin podešen kao **INPUT**, inače ova funkcija neće raditi kako treba.

Funkcija `digitalRead(pin)` čita vrijednost sa zadanog digitalnog pina i vraća rezultat **HIGH** ili **LOW**. Funkciju pozivamo unutar funkcije **loop()**:

```cpp

void loop() {
  
  //digitalRead() is a function that reads the value from a specified digital pin, either HIGH or LOW.
  bool reading = digitalRead(BUTTON_PIN);
}
```

## Debounce tipkala

Kad pritisneš tipkalo, ono iz otvorenog sklopa prelazi u zatvoreni i to doživljavamo kao jedan, čist pritisak. Nama se čini kao jedna radnja jer smo mnogo sporiji od elektronike. Mikrokontroler, koji stanje tipkala provjerava na tisuće puta u tom kratkom trenutku, vidi nešto drugo.

Dok pritišćeš tipkalo, dvije metalne površine unutra se dodiruju. Kako nisu savršeno glatke ni poravnate, mogu se u vrlo kratkom vremenu nekoliko puta spojiti i razdvojiti prije nego što se potpuno ustale. Za mikrokontroler to izgleda kao da je tipkalo pritisnuto više puta zaredom, iako si ga ti pritisnuo samo jednom.

<CenteredImage src="/img/nula-beginner-kit-arduino/button-debouncing-graph.png" alt="Debounce tipkala prikazan grafički" caption="Grafički prikaz debouncea tipkala" />

Najjednostavnija softverska strategija za rješavanje toga je debouncing uz vremensku odgodu.  
Cilj je pričekati da prođe razdoblje „poskakivanja“ kontakata. Tu nam dobro dođe funkcija `millis()`.

Funkcija `millis()` vraća broj milisekundi koje su prošle od trenutka kada je pločica pokrenula trenutni program. Možemo je iskoristiti da provjerimo je li prošlo dovoljno vremena između zadnje promjene stanja tipkala i trenutnog vremena. Ako jest, spremni smo prihvatiti sljedeći pritisak. Ako nije, još malo čekamo.

## Promjena stanja LED diode

Svaki put kad otkrijemo ispravan, „čist“ pritisak, mijenjamo stanje LED diode pomoću jednostavnog preklapanja (toggle). Taj dio koda nalazi se unutar debounce logike.

```cpp
if (lastState == LOW && reading == HIGH) {

      //We toggle the ledState
      ledState = !ledState;

      //We turn the LED on or off depending on the current ledState                
      if (ledState == true) {
        digitalWrite(LED_PIN, HIGH);
      } else {
        digitalWrite(LED_PIN, LOW);
      }
    }
    lastState = reading;
```

## Kod

U nastavku je prikazan puni primjer koda za upravljanje LED diodom pomoću tipkala s debounceom. Koristi funkciju `millis()` kako bi provjerio je li prošlo dovoljno vremena između promjena stanja tipkala.

```cpp
/*
This is a variable to which we pass the number of pin that we had connected the BUTTON to.
The NULA board has a pin naming logic as follows: IO19, where 19 is the number that we give to the variable.
If you wish to use a different pin, make sure you are using a IO__ marked pin. 
*/
const int BUTTON_PIN = 19;

/*
This is a variable to which we pass the number of pin that we had connected the LED to. 
The NULA board has a pin naming logic as follows: IO4, where 4 is the number that we give to the variable.
If you wish to use a different pin, make sure you are using a IO__ marked pin. 
*/
const int LED_PIN = 4;

/*
Those are the variables used for button debouncing. To learn more about button debouncing, check out our example
documentation at: <link placeholder>
*/
bool ledState = false;
bool lastState = LOW;
unsigned long lastChangeMs = 0;
const unsigned long debounceMs = 25;

void setup() {

  /*
  pinMode() is a function that configures the specified pin to behave either as an input or in this case as an output.
  This simply means that the pin "listens" for the available data when in input mode, and "writes" data when in output mode.
  As our pin needs to detect if the button has been pressed, we will put the pin in INPUT mode.
  */
  pinMode(BUTTON_PIN, INPUT);  

  /*
  pinMode() is a function that configures the specified pin to behave either as an input or in this case as an output.
  This simply means that the pin "listens" for available data when in input mode, and "writes" data when in output mode.
  As our pin needs to turn on the LED, we will put the pin in OUTPUT mode.
  */
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  
  //digitalRead() is a function that reads the value from a specified digital pin, either HIGH or LOW.
  bool reading = digitalRead(BUTTON_PIN);

  /*
  millis() is a function that returns the number of milliseconds passed since the board began running the current program.
  In this example, we use this function to check out if the debouncing period has finnished. This number will overflow 
  (go back to zero), after approximately 50 days.
  */
  unsigned long now = millis();
 
  /*
  This is our debouncing logic, we check is the button has been pressed and if enough time has passed so that we dont get 
  false readings because of the noise in the signal.
  */
  if (reading != lastState && (now - lastChangeMs) > debounceMs) {
    lastChangeMs = now;

    /*
    As we are using the pull-down method for connecting the button, we need to toggle the LED when the button gets pressed,
    in other words, when the button state goes from LOW to HIGH.
    */
    if (lastState == LOW && reading == HIGH) {

      //We toggle the ledState
      ledState = !ledState;

      //We turn the LED on or off depending on the current ledState                
      if (ledState == true) {
        digitalWrite(LED_PIN, HIGH);
      } else {
        digitalWrite(LED_PIN, LOW);
      }
    }
    lastState = reading;
  }
}

```
